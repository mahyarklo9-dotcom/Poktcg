# ربات تلگرامی کارت‌های پوکمون (Pokémon TCG Bot)

ربات تلگرامی ساخته‌شده با Aiogram 3 برای جمع‌آوری، خرید، فروش و معامله کارت‌های پوکمون
با استفاده از دیتابیس آنلاین [TCGdex](https://tcgdex.dev).

## ساختار پروژه

| فایل | کار آن |
|---|---|
| `bot.py` | فایل اصلی ربات؛ همه دستورها اینجا تعریف شده‌اند |
| `db.py` | ساخت جدول‌های دیتابیس (users, inventory, market, trades) |
| `economy.py` | مدیریت امتیاز کاربران، کول‌داون `/mony` و خرید پک |
| `inventory.py` | افزودن/حذف/انتقال کارت در اینونتوری کاربر |
| `market.py` | بازار خرید و فروش کارت بین کاربران |
| `trade.py` | معامله مستقیم بین دو کاربر با تایید دوطرفه |
| `cards_api.py` | اتصال به API عمومی TCGdex برای دریافت کارت‌های تصادفی و اطلاعاتشان |

## دستورهای ربات

- `/start` `/help` `/balance` — معرفی و وضعیت حساب
- `/mony` — دریافت ۳۰ تا ۱۲۰ امتیاز رایگان (هر ۱ ساعت یک‌بار)
- `/shop` — خرید یک پک با ۳۰ امتیاز
- `/open` — باز کردن یک پک و دریافت ۸ کارت تصادفی
- `/inv` — مشاهده اینونتوری
- `/sell <نام کارت> <قیمت>` — گذاشتن کارت در بازار
- `/market` — مشاهده آگهی‌های فعال بازار
- `/buy <شماره آگهی>` — خرید کارت از بازار
- `/cancel <شماره آگهی>` — لغو آگهی فروش خودت
- `/trade <کارت من> / <کارت طرف>` — پیشنهاد معامله (باید روی پیام طرف مقابل ریپلای کنی)؛ طرف مقابل با دکمه «قبول» یا «رد» پاسخ می‌دهد

## اجرای محلی (تست روی سیستم خودت)

```bash
python -m venv venv
source venv/bin/activate        # ویندوز: venv\Scripts\activate
pip install -r requirements.txt

cp env.example .env
# حالا BOT_TOKEN را داخل فایل .env با توکن ربات‌ت از @BotFather جایگزین کن

python bot.py
```

## استقرار روی Railway

1. یک پروژه جدید در [Railway](https://railway.app) بساز و این پوشه را به‌عنوان ریپو وصل کن (یا با Railway CLI آپلود کن).
2. در تنظیمات پروژه، متغیر محیطی `BOT_TOKEN` را با توکن واقعی ربات‌ت تنظیم کن.
3. فایل‌های `Procfile` و `railway.json` از قبل آماده‌اند و دستور اجرا (`python bot.py`) را مشخص کرده‌اند؛ نیازی به تنظیم دستی نیست.
4. دیپلوی را بزن؛ Railway به‌صورت خودکار `requirements.txt` را نصب و ربات را اجرا می‌کند.

> نکته: دیتابیس روی `database.db` (SQLite) ذخیره می‌شود. در Railway بدون Volume دائمی، این فایل با هر دیپلوی جدید ریست می‌شود؛
> برای نگه‌داشتن داده‌ها بین دیپلوی‌ها، یک Volume به سرویس وصل کن و مسیر آن را به جایی که `database.db` ساخته می‌شود اشاره بده.

## نکات فنی

- لیست کارت‌ها از API عمومی TCGdex با کش یک‌ساعته خوانده می‌شود تا هر بار `/open` باعث دانلود کل دیتابیس نشود.
- اگر API موقتاً در دسترس نباشد، پکی که هنگام `/open` مصرف شده بود به‌صورت خودکار به کاربر برگردانده می‌شود.
- معامله (`/trade`) فقط در گروه‌ها/چت‌هایی کار می‌کند که بتوانی روی پیام طرف مقابل ریپلای بزنی.
